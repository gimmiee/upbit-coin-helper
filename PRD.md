# PRD — 코린이 구조대 (Coin Helper)

## 1. 문서 정보

| 항목 | 내용 |
|------|------|
| 제품명 | 코린이 구조대 |
| 데이터 소스 | Upbit Open API (일봉·분봉 캔들, 마켓 목록 등) |
| 문서 목적 | 제품이 무엇을 하는지, 누구를 위한 것인지, 범위와 비범위를 명확히 함 |

---

## 2. 한 줄 요약

암호화폐(업비트 KRW 마켓 중심)를 선택하고 기간을 정하면, **가격·거래량·변동률**을 차트로 보여 주고, 같은 데이터를 바탕으로 **초보자용 요약 카드**와 **상세 지표**를 함께 제공하는 **정보용 분석 웹앱**이다.

---

## 3. 문제 정의 & 타겟 사용자

### 3.1 문제

- 차트와 지표는 많은데, **무엇을 어떤 순서로 보면 되는지** 정리하기 어렵다.
- 여러 화면을 오가며 **같은 기간·같은 코인** 기준으로 맞추기 번거롭다.

### 3.2 타겟

- 업비트에서 코인을 보기 시작한 **초·중급 사용자(코린이)**.
- **투자 자문**이 아니라, **스스로 판단할 때 참고할 수 있는 요약**이 필요한 사람.

---

## 4. 목표 & 비목표

### 4.1 목표 (In Scope)

- 단일 화면에서 **조회 → 상태/요약 → (선택) 상세 지표 → 차트** 흐름 제공.
- **일봉 기반** 분석: RSI, MA5/MA20, 구간 내 가격 위치, 변동성, 거래량 압력, Force(가격변화×거래량), 매물대 스타일 목표/손절 참고선, 가짜 하락 휴리스틱, BTC 동조화(알트), 거래량-가격 다이버전스 등.
- **실시간(분봉) 차트**: 타임프레임·캔들 수, 줌/팬, 목표가·손절가 표시 옵션.
- **다크/라이트 테마** 전환.
- **반응형 UI**: 모바일에서도 조회·읽기 가능하도록 레이아웃 대응.

### 4.2 비목표 (Out of Scope)

- **매수/매도 지시**, 수익 보장, 개인별 맞춤 포트폴리오 추천.
- **주문 실행**, 잔고·API 키 연동 거래.
- **법적·세무·투자 적합성** 판단.
- Upbit API **이용약관·레이트리밋**을 대신 준수해 주는 것(클라이언트/배포 환경에 따름).

---

## 5. 사용자 시나리오 (핵심 플로우)

1. 페이지 진입 → 마켓 목록 로드.
2. 코인 검색/선택, **기간 프리셋** 또는 **시작일–종료일** 선택.
3. **조회** 클릭 → 일봉 데이터 수집 후 대시보드 갱신.
4. **오늘의 코린이 맞춤 리포트**로 한 줄 결론·근거·인사이트 확인.
5. **퀵 카드**로 가격대, RSI, 변동성, 탈출(목표/손절 참고), 가짜 하락, BTC 눈치(알트) 확인.
6. 필요 시 **전문가 상세 지표** 펼쳐 숫자·근거 확인.
7. **가격/거래량/변동률 차트** 및 **실시간 분봉 차트**로 시각적 확인.

## 6. 기능 요구사항

### 6.1 데이터 & API

| ID | 요구사항 | 비고 |
|----|-----------|------|
| F-1 | 업비트 **일봉** 캔들 조회 (마켓, 개수, 기간) | 프록시 경로: `/api/upbit/candles` |
| F-2 | 업비트 **분봉** 캔들 조회 (실시간 차트) | 동일 핸들러, `unit` 파라미터 |
| F-3 | **마켓 전체 목록** 조회 (검색/셀렉트용) | `/api/upbit/market-all` |
| F-4 | WebSocket으로 **실시간 현재가** 등 보조 스트림 (구현된 경우) | `app.js` 내 WS URL |

### 6.2 분석·표시 (일봉 기준)

| ID | 요구사항 |
|----|-----------|
| A-1 | 최근 **30일** 고저 대비 **현재가 위치(%)** — 퀵 카드는 20/80% 등 임계, 상세는 구간 라벨(바닥/저점/중립/고점/천장 등) |
| A-2 | **RSI(14)** — 70 이상 과열, 30 이하 과매도 등 문구 |
| A-3 | **변동성 레짐** — 평균 일변동·일중 변동폭 조합으로 고/저/보통 |
| A-4 | **변동성 돌파** — 당일 변동폭 vs 직전 5일 평균 비율(2배 이상 등) |
| A-5 | **MA5 / MA20**, 기울기·골든/데드크로스 표시 |
| A-6 | **Force**(전일 대비 가격차 × 거래량) 및 7일 평균, 미터 정규화 |
| A-7 | **모멘텀** — 최근 7일 vs 직전 7일 수익률 차이 |
| A-8 | **거래량 압력** — 최근 7일 vs 직전 7일 평균 거래량 비율 |
| A-9 | **매물대 기반 목표가·손절가(참고)** — 30일 룩백, 10구간, 최소 이격률 등 (코드 상 상수) |
| A-10 | **가짜 하락(휴리스틱)** — 하락 폭 임계(5일 평균 변동 기반) + 거래량 5일 평균 대비 비율 |
| A-11 | **BTC 동조화** — 알트 선택 시 일간 수익률 상관, 카드/전문가 패널별 구간 기준 상이 가능 |
| A-12 | **거래량-가격 다이버전스** — N일 평균 대비 M배 이상 + 방향에 따른 신호 |
| A-13 | **맞춤 리포트 결론** — RSI·MA 기울기·BTC 상관 클래스·Force 부호 등 **점수형 휴리스틱** |

### 6.3 UI/UX

| ID | 요구사항 |
|----|-----------|
| U-1 | 상단 **면책·API 출처** 고지 유지 |
| U-2 | **모바일**에서 컨트롤·카드 세로 스택, 터치 영역·가로 스크롤(범례·바 등) 고려 |
| U-3 | 차트 범례 **토글/툴팁** (구현된 범위 내) |

### 6.4 빌드·배포

| ID | 요구사항 |
|----|-----------|
| D-1 | `pnpm build` → 정적 산출물 (`vite build`) |

## 7. 비기능 요구사항

- **성능**: 조회 시 불필요한 중복 요청 최소화(구현 수준에 따름).
- **접근성**: 버튼·토글에 `aria-*` 등 기본 대응(구현된 범위).
- **투명성**: 지표는 **휴리스틱**이며 투자 결과를 보장하지 않음을 UI에 명시.

---

## 8. 성공 지표 (제품 관점, 참고)

- 사용자가 **한 번의 조회**로 기간·코인 기준이 맞는 요약과 차트를 얻는다.
- 초보자용 카드만으로도 **다음에 무엇을 더 볼지** 방향이 잡힌다(정성 피드백).

---

## 9. 리스크 & 가정

- Upbit API **변경·장애·제한** 시 일부 기능이 실패할 수 있음.
- **과거 데이터 기반** 지표는 미래를 예측하지 않음.
- 동조화·상관은 **표본 기간·정렬 방식**에 민감함.

---

## 10. 용어

- **코린이**: 암호화폐 투자 초보 사용자를 가리키는 비공식 표현(서비스 톤).
- **PRD**: 본 문서 — 실제 구현은 저장소의 `app.js`, `index.html`, API 핸들러를 기준으로 한다.
